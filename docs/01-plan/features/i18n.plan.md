# Feature Plan: i18n (KO/EN locale switcher + Shell coverage)

## Executive Summary

| 관점 | 내용 |
|------|------|
| Problem | next-intl 인프라(`/ko`·`/en` 라우팅, request/proxy)는 깔려있지만 ① 사용자가 언어를 전환할 UI가 없고 ② Shell(Header/Rail/BottomNav) 텍스트가 대부분 하드코딩 한국어 — 영어 사용자가 사실상 진입 불가능 |
| Solution | "더보기" 메뉴를 신설(웹 Rail 하단 / 모바일 프로필 내)하고 그 안에 KO\|EN 토글 + Shell 영역 메시지를 ko·en JSON으로 추출 + hreflang/locale별 metadata 적용 |
| UX Effect | 영어 사용자도 1탭으로 네비게이션 영어 사용 가능, 선택은 URL prefix + 쿠키로 기억. 검색엔진은 두 언어 페이지를 올바르게 인덱싱 |
| Core Value | "ClipClef은 한국 큐레이터만의 도구가 아니다" — 깔려있던 i18n 자산을 사용자에게 처음으로 노출, 글로벌 진입 첫 발판 |

## Context Anchor

| | |
|--|--|
| WHY | 영어 사용자가 첫 화면에서 언어를 바꿀 방법이 없어 진입 자체가 막혀 있음. next-intl + /ko·/en 라우팅은 이미 갖춰져 있어 UI/커버리지만 채우면 즉시 활성화되는 "잠자는 자산" 상태 |
| WHO | 영어권 잠재 사용자(1차) + 한국어 사용자(토글 UX 노출 대상). 큐레이터/리스너 모두 |
| RISK | 중간-낮음. 인프라(next-intl) 무변경. Rail "설정" 아이콘을 "더보기"로 교체(최근 commit `da2ce61` 직후 재교체)하므로 자산 정리 필요. 메시지 키 누락 시 throw 가능성 |
| SUCCESS | (1) Rail 더보기 + 모바일 프로필 더보기 안에 KO\|EN 토글 동작 (2) Shell(Header/Rail/BottomNav) 하드코딩 한국어 0건 (3) /en 진입 시 `<html lang="en">` + hreflang + 영문 metadata (4) 토글 시 URL prefix + `NEXT_LOCALE` 쿠키 동시 갱신 |
| SCOPE | Shell only. 페이지 본문(Home/Search/Filter/Playlist/Comment 등)·동적 콘텐츠·자동 감지·영문 OG 이미지·알림/이메일은 follow-up |

## 1. 요구사항

### 1.1 "더보기" 메뉴 신설 (UI)

**웹 (DesktopRail 하단)**
- 현재: `/Settings.svg` 아이콘 + 라벨 "설정 (준비 중)" (최근 commit `da2ce61`로 교체됨)
- 변경: "더보기" 아이콘으로 교체, 클릭 시 팝오버/메뉴 표시
- 메뉴 항목: 기본정보 · 내좋아요 · 내댓글 · 내큐레이션 · KO\|EN

**모바일 (`/me` 프로필 페이지)**
- 프로필 페이지 안에 "더보기" 섹션(또는 하위 페이지) 추가
- 메뉴 항목 동일: 기본정보 · 내좋아요 · 내댓글 · 내큐레이션 · KO\|EN
- BottomNav 자체는 변경 없음 (4-zone 라우팅 유지)

**KO\|EN 토글 컴포넌트**
- 텍스트 토글 형태 (`KO | EN`, 활성 측 강조)
- 클릭 시: ① `useRouter().replace()`로 현재 path의 locale prefix 교체 ② `NEXT_LOCALE` 쿠키 set (1년)
- 메뉴 안에서 한 줄 항목으로 노출

### 1.2 Shell 메시지 커버리지

대상 영역 (하드코딩 한국어 추출):
- `src/components/layout/Header.tsx` — 검색 placeholder, 뒤로가기, 홈으로, "전체 결과 보기"
- `src/components/layout/DesktopRail.tsx` — 주 메뉴, 홈으로, 만들기, 프로필, 더보기 (+ 옛 "설정 (준비 중)" 라벨 정리)
- `src/components/layout/MobileBottomNav.tsx` — 만들기, 프로필
- 공통 버튼/공통 라벨: 저장, 취소, 로딩, 오류, 로그인/로그아웃, 뒤로 (이미 `common.*` 일부 존재)

처리 방식:
- `src/i18n/ko.json` / `src/i18n/en.json`에 `nav.*`, `more.*`, `header.*` 네임스페이스 추가
- 각 파일에서 `useTranslations()` 도입, 하드코딩 문자열 제거
- aria-label도 동일 처리 (스크린리더용)

### 1.3 Persistence

- **URL prefix**: `/ko/...`, `/en/...` (next-intl 기본 — 이미 동작)
- **쿠키**: `NEXT_LOCALE=ko|en`, `Path=/`, `Max-Age=1y`, `SameSite=Lax`
- **우선순위**: URL prefix > 쿠키 > 기본값(`ko`)
- 토글 클릭 시 쿠키 set + URL replace 동시 실행

### 1.4 SEO

- `src/app/[locale]/layout.tsx`의 `generateMetadata()`에 `alternates.languages` 추가
  - `ko: '/ko/<path>'`
  - `en: '/en/<path>'`
  - `x-default: '/ko/<path>'`
- locale별 `title` / `description` 분리 (`SITE_DESCRIPTION_KO` / `SITE_DESCRIPTION_EN`)
- `<html lang={locale}>` 적용 (현재 정적이면 동적으로)
- `sitemap.ts`에 ko/en 두 URL 모두 출력

## 2. 비범위 (Out of Scope)

- 페이지 본문 영어화 (Home/Search/Filter/Playlist/Comment 페이지 본문)
- 동적 콘텐츠 번역 (플레이리스트 제목, 댓글, 유튜브 메타데이터)
- Accept-Language 자동 감지 (1차는 명시적 토글만)
- 영문 OG 이미지 (영문 og-default.png 빌드)
- 알림/이메일 영어화
- 어드민 페이지 영어화

## 3. 영향도

**파일 추가**
- `src/components/i18n/LocaleToggle.tsx` — KO\|EN 텍스트 토글 (path replace + cookie set)
- `src/components/layout/MoreMenu.tsx` — 웹 Rail용 더보기 팝오버 (메뉴 컨테이너)
- `src/features/profile/components/ProfileMore.tsx` 또는 `/me/more/page.tsx` — 모바일 더보기 섹션/페이지 (Design 단계 결정)

**파일 변경**
- `src/components/layout/DesktopRail.tsx` — 설정 아이콘 → 더보기, MoreMenu 연결
- `src/components/layout/Header.tsx` — useTranslations 도입
- `src/components/layout/MobileBottomNav.tsx` — useTranslations 도입
- `src/app/[locale]/me/page.tsx` — 더보기 섹션 진입점
- `src/app/[locale]/layout.tsx` — `<html lang>` + `generateMetadata` alternates
- `src/i18n/ko.json` / `src/i18n/en.json` — Shell 네임스페이스 보강
- `src/lib/seo.ts` — locale별 description 분리
- `src/app/sitemap.ts` (있으면) — 두 locale URL 출력

**자산**
- `/public/Settings.svg` — 더보기로 교체 후 unused가 되면 제거 (또는 더보기 아이콘으로 재활용 결정)

## 4. Success Criteria

- [ ] Rail 하단 아이콘이 "더보기"로 표시되며 클릭 시 메뉴 5개 항목 노출
- [ ] 모바일 /me 페이지에서 더보기 섹션 진입 가능, 동일 5개 항목
- [ ] KO\|EN 토글 클릭 시 ① URL이 `/en/...` ↔ `/ko/...` 로 즉시 전환 ② `NEXT_LOCALE` 쿠키가 갱신됨
- [ ] Shell(Header/Rail/BottomNav)에서 정규식 `[가-힣]` 매칭 0건 (Plan 1.2 대상 범위)
- [ ] `<html lang>` 속성이 현재 locale과 일치
- [ ] DOM `<head>` 에 `<link rel="alternate" hreflang="ko|en|x-default">` 출력
- [ ] /en 페이지 `<title>` / `<meta name="description">`이 영문
- [ ] 외부에서 / 첫 진입 시: URL prefix > 쿠키 > 기본 ko 순으로 결정
- [ ] sitemap.xml에 `/ko/...` `/en/...` 두 URL 출력 (해당 페이지 한정)
