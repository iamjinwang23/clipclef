### 18. iOS Safari 필터 칩 터치 불응 버그 수정

**근본 원인 분석**
- FilterBar 스크롤 컨테이너에 `scroll-snap-type: x mandatory` 적용
- iOS Safari는 snap scrolling 영역의 모든 touch를 scroll 판별을 위해 가로채므로 내부 tap 이벤트가 일부 억제됨
- 필터 칩(FilterChip, EraFilter)은 이 컨테이너 내부에 있어 tap이 인식되지 않음

**시도 경과**
1. 1차: `onMouseDown` → `onClick` 변경 + `touchstart` 외부 클릭 핸들러 추가 — 불충분
2. 2차: `touch-action: manipulation` 추가 — 불충분
3. **최종**: `scrollSnapType: 'x mandatory'` 제거 — 근본 원인 해소

**변경 파일**
- `src/features/filter/components/FilterBar.tsx` — scroll 컨테이너 style에서 `scrollSnapType` 제거

> 참고: `scroll-snap-type`은 캐러셀 등 의도적 스냅에만 사용. 필터 칩 row는 자유 스크롤이 적합.

- `src/components/layout/Header.tsx` — `handleLogin`에 인앱 guard 추가

**인앱 감지 패턴 (`browser.ts`)**
```ts

/NAVER|KAKAOTALK|Line\/|FB_IAB|FBAV|FBAN|Instagram|Snapchat|Twitter/
```

### 37. 모바일 알림 페이지 수정 (2026-04-15)

**수정 내역**

1. **`<a><button>` 중첩 에러** — `NotificationItem`이 `<Link>` 안에서 `<button>`을 렌더링해 무효 HTML 발생.
   - `onClick` prop 없을 때 `<div>`로 렌더링하도록 분기 처리.

2. **서버 컴포넌트 세션 만료 redirect** — `proxy.ts`의 `setAll`이 `request.cookies`를 업데이트하지 않아 Supabase 세션 갱신 후에도 서버 컴포넌트의 `getUser()`가 만료 토큰을 읽고 `null` 반환 → 홈 redirect.
   - `setAll`에서 `pendingCookies.push` 외에 `request.cookies.set`도 호출하도록 수정.

3. **`middleware.ts` 빌드 에러** — Next.js 16에서 `middleware.ts`는 `proxy.ts`로 대체됨. `middleware.ts`를 새로 생성하면 `proxy.ts`와 충돌하여 빌드 실패.
   - 잘못 추가한 `src/middleware.ts` 삭제.

| **proxy.ts setAll 패턴** | `setAll`에서 `request.cookies.set`을 호출해야 서버 컴포넌트가 갱신된 세션 토큰을 읽을 수 있음. `pendingCookies.push`만 하면 response에는 반영되지만 서버 컴포넌트에서 여전히 만료 토큰 읽음. | `setAll`에서 `toSet.forEach(({ name, value }) => request.cookies.set(name, value))` 반드시 포함 |
| 인앱 브라우저 로그인 | 네이버앱 등 WebView에서 Google OAuth 차단 | 사용자에게 Safari/Chrome 안내 (코드 우회 불가) |
| Vercel alias drift | `--prod` 배포 후 `clipclef.vercel.app` alias 수동 재설정 필요 | Vercel Dashboard → Domains에 `clipclef.vercel.app` 등록 |
| YouTube 재인증 | 기존 로그인 유저는 토큰 DB 저장을 위해 1회 재인증 필요 | 재인증 유도 UI 이미 구현됨 |
