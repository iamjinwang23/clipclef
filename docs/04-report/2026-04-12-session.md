### 18. iOS Safari 필터 칩 터치 불응 버그 수정

**근본 원인 분석**
- FilterBar 스크롤 컨테이너에 `scroll-snap-type: x mandatory` 적용
- iOS Safari는 snap scrolling 영역의 모든 touch를 scroll 판별을 위해 가로채므로 내부 tap 이벤트가 일부 억제됨
- 필터 칩(FilterChip, EraFilter)은 이 컨테이너 내부에 있어 tap이 인식되지 않음

**시도 경과**
1. 1차: `onMouseDown` → `onClick` 변경 + `touchstart` 외부 클릭 핸들러 추가 — 불충분
2. 2차: `touch-action: manipulation` 추가 — 불충분
3. **최종**: `scrollSnapType: 'x mandatory'` 제거 — 근본 원인 해소

**변경 파일**
- `src/features/filter/components/FilterBar.tsx` — scroll 컨테이너 style에서 `scrollSnapType` 제거

> 참고: `scroll-snap-type`은 캐러셀 등 의도적 스냅에만 사용. 필터 칩 row는 자유 스크롤이 적합.

### 19. 인앱 브라우저 Google OAuth 오류 대응

**배경**
- 네이버앱, 카카오톡, Instagram 등 인앱 브라우저에서 Google OAuth 시도 시
  `disallowed_useragent` (403) 에러 발생
- Google 정책: 서드파티 앱 WebView에서 OAuth 차단 (보안 이유)
- 코드 레벨로 우회 불가 — 사용자를 외부 브라우저로 안내하는 것이 유일한 해결책

**변경 파일**
- `src/lib/browser.ts` (신규) — 인앱 브라우저 감지 유틸
- `src/features/interaction/components/LoginModal.tsx` — 인앱 시 외부 브라우저 안내 메시지
- `src/components/layout/Header.tsx` — `handleLogin`에 인앱 guard 추가

**인앱 감지 패턴 (`browser.ts`)**
```ts
// 알려진 인앱 브라우저 UA 키워드
/NAVER|KAKAOTALK|Line\/|FB_IAB|FBAV|FBAN|Instagram|Snapchat|Twitter/

// iOS WebView: AppleWebKit 있으나 Safari 토큰 없음
/iPhone|iPad|iPod/ && /AppleWebKit/ && !/Safari/
```

**UX 처리**
- LoginModal: 인앱 감지 시 Google 로그인 버튼 대신 "Safari 또는 Chrome에서 접속해 주세요" 안내
- Header: 인앱 감지 시 `alert()` 후 OAuth 호출 차단

> YouTube(Google 자사 앱)가 인앱 브라우저에서 로그인 가능한 이유:
> Google 계열 앱에는 `allowlist` 예외 적용. 서드파티 앱(Supabase/Vercel 도메인)은 해당 없음.

### 20. Vercel 배포 alias 관리

**문제**: `vercel --prod` 는 새 URL을 생성하지만 수동으로 설정한 alias(`clipclef.vercel.app`)는 자동 업데이트되지 않음

**해결 패턴**
```bash
npx vercel --prod
npx vercel alias set [new-deployment-url] clipclef.vercel.app
```

**장기 해결책**: Vercel 대시보드 → Project Settings → Domains에 `clipclef.vercel.app` 등록 시 `--prod` 배포마다 자동 연결
