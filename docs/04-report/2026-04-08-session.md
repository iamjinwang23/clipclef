### 8. YouTube 내보내기 인증 수정

**문제**
- `session.provider_token`은 OAuth 콜백 직후에만 존재
- 이후 `getSession()` 호출 시 항상 `null` → 내보내기 실패

**해결 — DB 토큰 저장 방식**

**변경 파일**
- `supabase/migrations/006_user_tokens.sql` (신규)
- `src/app/api/auth/callback/route.ts` — OAuth 콜백에서 `provider_token` → `user_tokens` 테이블 저장
- `src/app/api/youtube/export/route.ts` — DB에서 토큰 읽기 + 만료 시 자동 refresh
- `.env.local` — `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET` 추가

**DB 변경**
```sql
CREATE TABLE user_tokens (
  user_id UUID PRIMARY KEY,
  provider TEXT,
  access_token TEXT,
  refresh_token TEXT,
  expires_at TIMESTAMPTZ,
  updated_at TIMESTAMPTZ
);
```

**동작 흐름**
1. Google OAuth 로그인 → 콜백에서 `provider_token` DB 저장
2. export API 호출 → DB에서 토큰 조회
3. 만료 시 `refresh_token`으로 자동 갱신 후 재저장
4. 갱신 불가 → `youtube_not_connected` 에러 → 재인증 유도

> ⚠️ 적용 전 필요: "YouTube 연결" 버튼으로 Google 재인증 1회 필요

### 14. 다크 테마 근본 수정

**근본 원인**
1. `color-scheme: dark` 미설정 → 브라우저 UI 요소(스크롤바, 폼) 라이트 렌더링
2. `<html>` 요소에 배경색 미적용 → 흰 배경 노출
3. Tailwind v4 + Next.js Turbopack 환경에서 CSS 미처리 시 폴백 없음

**수정 (`globals.css`)**
```css
:root {
  color-scheme: dark;   /* 신규 */
  --background: #0D0D0D;
  ...
}
html { background-color: #0D0D0D; color: #F0F0F0; }  /* 신규 */
body { background-color: #0D0D0D; color: #F0F0F0; }  /* var() → 하드코딩 */
```

**수정 (`[locale]/layout.tsx`)**
```tsx
export const viewport = { colorScheme: 'dark' };  // <meta name="color-scheme"> 자동 생성

<html style={{ colorScheme: 'dark', backgroundColor: '#0D0D0D' }}>
<body style={{ backgroundColor: '#0D0D0D', color: '#F0F0F0' }}>
```

**효과**: Tailwind CSS 처리 여부·환경과 무관하게 다크 배경 보장
