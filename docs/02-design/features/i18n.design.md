# Feature Design: i18n (KO/EN locale switcher + Shell coverage)

> Architecture: **Option C — Pragmatic Balance**
> Plan: `docs/01-plan/features/i18n.plan.md`

## Context Anchor

| | |
|--|--|
| WHY | 영어 사용자가 첫 화면에서 언어를 바꿀 방법이 없어 진입 자체가 막혀 있음. next-intl + /ko·/en 라우팅은 이미 갖춰져 있어 UI/커버리지만 채우면 즉시 활성화되는 "잠자는 자산" 상태 |
| WHO | 영어권 잠재 사용자(1차) + 한국어 사용자(토글 UX 노출 대상). 큐레이터/리스너 모두 |
| RISK | 중간-낮음. 인프라(next-intl) 무변경. Rail "설정" 아이콘을 "더보기"로 교체(최근 commit `da2ce61` 직후 재교체)하므로 자산 정리 필요. 메시지 키 누락 시 throw 가능성 |
| SUCCESS | (1) Rail 더보기 + 모바일 프로필 더보기 안에 KO\|EN 토글 동작 (2) Shell(Header/Rail/BottomNav) 하드코딩 한국어 0건 (3) /en 진입 시 `<html lang="en">` + hreflang + 영문 metadata (4) 토글 시 URL prefix + `NEXT_LOCALE` 쿠키 동시 갱신 |
| SCOPE | Shell only. 페이지 본문 / 동적 콘텐츠 / 자동 감지 / 영문 OG / 알림·이메일은 follow-up |

## 1. Overview

기존 `ProfileSettingsMenu`(이미 5개 항목 모두 구현됨)를 **`MoreMenu`** 로 일반화하여 위/아래 양방향 팝오버를 지원하게 만들고, KO\|EN 토글만 별도 컴포넌트(**`LocaleToggle`**)로 분리해 메뉴 마지막 row에 슬롯으로 끼운다. 이 메뉴는 두 진입점에서 호출된다:

- **Web**: `DesktopRail` 하단의 "설정 (준비 중)" 자리 → "더보기" 트리거로 교체. 위로 열림.
- **Mobile**: 기존 `/me/profile` 페이지의 ⋯ 버튼 그대로(이미 동작 중). 아래로 열림.

Shell 영역(Header/Rail/BottomNav) 하드코딩 한국어를 `useTranslations` 기반으로 추출하고, layout.tsx의 metadata에 locale별 description을 적용한다.

## 3. Components

### 3.1 `LocaleToggle.tsx` (신규)

**Props**: `{ className?: string }` (메뉴 안 row 스타일 상속)

**책임**:
- 현재 `useLocale()`로 활성 언어 표시
- 클릭(또는 키보드) 시:
  1. 새 locale로 path 재구성 — `/ko/foo/bar` ↔ `/en/foo/bar`
  2. `document.cookie = 'NEXT_LOCALE=<new>; Path=/; Max-Age=31536000; SameSite=Lax'`
  3. `router.replace(newPath)` (스크롤/state 보존)
- 외관: `KO` | `EN` 두 텍스트, 활성 측 강조(`text-foreground font-semibold`), 비활성(`text-subtle`). 버튼 div 1개로 toggle.

**구현 스케치**:
```tsx
'use client';
import { useLocale, useTranslations } from 'next-intl';
import { usePathname, useRouter } from 'next/navigation';
import { useTransition } from 'react';

const LOCALES = ['ko', 'en'] as const;
type Locale = typeof LOCALES[number];

export function LocaleToggle({ className }: { className?: string }) {
  const locale = useLocale() as Locale;
  const pathname = usePathname();
  const router = useRouter();
  const t = useTranslations('more');
  const [isPending, startTransition] = useTransition();

  const setLocale = (next: Locale) => {
    if (next === locale) return;
    // /ko/foo → /en/foo
    const newPath = pathname.replace(/^\/(ko|en)/, `/${next}`) || `/${next}`;
    document.cookie = `NEXT_LOCALE=${next}; Path=/; Max-Age=31536000; SameSite=Lax`;
    startTransition(() => router.replace(newPath));
  };

  return (
    <div className={`flex items-center justify-between px-4 py-2 ${className ?? ''}`}>
      <span className="text-sm text-[var(--foreground)]">{t('language')}</span>
      <div className="flex items-center gap-1 text-sm" aria-disabled={isPending}>
        {LOCALES.map((lc, i) => (
          <span key={lc} className="contents">
            {i > 0 && <span className="text-[var(--subtle)]">|</span>}
            <button
              type="button"
              onClick={() => setLocale(lc)}
              className={lc === locale
                ? 'font-semibold text-[var(--foreground)]'
                : 'text-[var(--subtle)] hover:text-[var(--foreground)]'}
            >
              {lc.toUpperCase()}
            </button>
          </span>
        ))}
      </div>
    </div>
  );
}
```

### 3.2 `MoreMenu.tsx` (rename from `ProfileSettingsMenu`)

**Props**:
```ts
interface Props {
  locale: string;
  direction?: 'down' | 'up';   // default 'down'
  align?: 'left' | 'right';    // default 'right'
  trigger?: ReactNode;         // default: ⋯ 점 3개 (기존 디자인)
```

**변경점 (vs 기존 ProfileSettingsMenu)**:
- 메뉴 항목 라벨을 `useTranslations('more')`로 교체:
  - `more.basicInfo` / `more.likes` / `more.comments` / `more.curations`
- 마지막 row에 `<LocaleToggle />` 삽입 (Link 5개 + Toggle 1개의 명확한 구분 — `<hr>` 또는 `border-t`)
- direction='up' 시 `bottom-full mb-2`, 'down' 시 `top-full mt-2`
- align='left' 시 `left-0`, 'right' 시 `right-0`

**메뉴 항목 구조**:
| Row | 컴포넌트 | path |
|-----|---------|------|
| 1 | Link | `/{locale}/me/settings` (기본 정보) |
| 2 | Link | `/{locale}/me/likes` (내 좋아요) |
| 3 | Link | `/{locale}/me/comments` (내 댓글) |

## 4. Data Flow

### 4.1 Locale 결정 우선순위 (proxy.ts → next-intl middleware)

```
요청 → next-intl middleware →
  1. URL prefix /ko or /en  → 그대로 사용
  2. 없음 + cookie NEXT_LOCALE → cookie 값으로 redirect
  3. 없음 + cookie 없음        → routing.defaultLocale ('ko')
```

이미 next-intl middleware가 위 동작을 기본 제공. 추가 코드 불필요.

### 4.2 토글 동작

```
사용자 KO|EN 클릭
  → LocaleToggle.setLocale(next)
  → cookie NEXT_LOCALE=next set
  → router.replace(/{next}/<rest>)
  → next-intl middleware 재실행 (URL prefix 일치 → pass)
  → RSC 재렌더, getMessages() 가 next.json 로드
  → useTranslations() 가 새 locale 메시지 반환
```

### 4.3 메시지 fallback

next-intl 기본: 키 없으면 `MISSING_MESSAGE` warning + 키 자체 출력. 본 작업은 ko/en 동시 추가하는 것이 원칙이며, fallback 헬퍼는 도입하지 않음 (누락 시 빠르게 발견하기 위함). 빌드 시 키 누락 검출은 follow-up.

## 6. SEO

### 6.1 `lib/seo.ts` 변경

```ts
const SITE_DESCRIPTIONS: Record<string, string> = {
  ko: '취향을 클립하고, 취향을 듣다 — 유튜브 플리 큐레이션',
  en: 'Clip your taste, listen to your taste — YouTube playlist curation',
};

export function getSiteDescription(locale: string): string {
  return SITE_DESCRIPTIONS[locale] ?? SITE_DESCRIPTIONS.ko;
}
```

기존 `SITE_DESCRIPTION` export는 유지(다른 곳 참조). `getSiteDescription`은 layout.tsx에서 사용.

### 6.2 `app/[locale]/layout.tsx` 변경

`generateMetadata` 안:
```ts
const description = getSiteDescription(locale);
return {
  // ...
  description,
  alternates: {
    canonical: `/${locale}`,
    languages: {
      ko: `${SITE_URL}/ko`,
      en: `${SITE_URL}/en`,
      'x-default': `${SITE_URL}/ko`,
    },
  },
  openGraph: {
    // ...
    description,
    locale: locale === 'ko' ? 'ko_KR' : 'en_US',
  },
  twitter: { /* ... */ description },
};
```

> 기존에 `languages[lc] = ${SITE_URL}/${lc}` 가 있긴 한데 `x-default`가 없음 → 추가. canonical은 `/${locale}` 유지.

### 6.3 `<html lang>` — 이미 동작

`<html lang={locale}>` 이미 layout.tsx에 있음. 변경 불필요.

### 6.4 sitemap

별도 작업 범위 — 현 sitemap.ts 가 이미 ko/en을 출력하면 보강 불필요. (Do 단계에서 확인)

## 7. 폴백 / 회귀 방지

| 항목 | 대응 |
|------|------|
| en.json 키 누락 | next-intl 기본 동작(콘솔 warn + 키 출력)으로 즉시 감지. 빌드 차단은 안 함. |
| 토글 시 URL이 `/ko` 또는 `/`인 root 경로 | regex `^\/(ko|en)` 가 매칭 안 되는 경우 `/${next}` fallback |
| 토글이 query string 잃음 | `pathname` + `useSearchParams()` 합쳐서 새 URL 생성 |
| Rail 기존 toast.info('준비 중입니다') | MoreMenu 연결 후 제거 (해당 라벨도 ko/en에서 삭제) |
| /Settings.svg 자산 | More.svg 자산이 없으면 inline SVG로 점 3개 horizontal 사용. Settings.svg는 `git rm`. |
